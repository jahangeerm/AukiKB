# Zapier’s AI Fluency Rubric

*Updated March 2026*

A grid for evaluating AI adoption: **six functions** (rows) × **four proficiency levels** (columns).

| Level | One-line framing |
|-------|------------------|
| **Unacceptable** | This is the extent of how I use AI at work. |
| **Capable** | I use AI to operate at a meaningfully higher level. |
| **Adoptive** | I orchestrate AI / build systems that elevate how I work. |
| **Transformative** | I re-engineer how work happens. |

---

## Engineering

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Uses AI as a lightweight assistant for snippets/summarization without changing core workflow. Cannot explain tool choices or demonstrate repeatable workflows. |
| **Capable** | Regularly uses AI for implementation, debugging, testing, and documentation with concrete examples of speed/quality gains. Shows tool literacy. |
| **Adoptive** | Defaults to AI-first approaches; builds workflows/tooling that improve output beyond themselves. Demonstrates judgment regarding tradeoffs and failure modes. |
| **Transformative** | Re-engineers how software is built so AI is part of the operating model (production, review, testing). Raises the bar for others by setting standards and building shared systems. |

---

## Product

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Uses AI for simple tasks like summarizing or looking up info, but output is “AI slop.” Work looks the same as it did before AI. |
| **Capable** | Has a structured approach for generating specs/prototypes. Uses AI to tap into user insights or data scale previously inaccessible. |
| **Adoptive** | Developed new skill sets (SQL, data analysis, dashboards) via AI. Built pipelines of agents for features. Has built “always-on” AI systems others rely on. |
| **Transformative** | The PM/Design role looks materially different than six months ago. Can show examples of abandoning the status quo to reinvent processes. Leading new ways of building (e.g., unlocking PMs to ship code). |

---

## Support

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Asks one-off questions then returns to manual work. Treats AI as a drafting shortcut for low-stakes communication only. |
| **Capable** | Has repeatable prompts for core job parts. Feeds relevant context into AI before complex work. Uses AI for tasks they previously wouldn’t have had time for. |
| **Adoptive** | Six months ago work was manual; now a connected set of tools handles it. Produces work others on the team use. Actively iterates on AI workflows as work changes. |
| **Transformative** | Changed what the team does, not just how fast. Categories of work no longer exist without human involvement. Influenced broader org frameworks and processes. |

---

## Marketing

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Uses AI for first drafts only; output reads like unedited AI. Uses AI for campaign ideas but has no real workflow. |
| **Capable** | Uses AI regularly across content, SEO, and performance review. Built a reusable prompt library for top content formats. |
| **Adoptive** | Has run AI-driven experiments with measurable results (e.g., 18% CTR increase). Built systems that draft, format, and schedule posts across channels. |
| **Transformative** | Built a personalization engine serving AI-generated variants at scale. Restructured how the team works and how success is measured. Automated entire job categories. |

---

## Sales & Revenue

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Uses AI summaries (e.g., Gong) after calls or for outreach emails. Cannot point to evidence that outreach is more effective or pipeline has improved. |
| **Capable** | Uses AI products built internally. Built a personal prospecting workflow that saves time and generates pipeline. Can position AI transformation strategies with customers. |
| **Adoptive** | Chains multiple AI tools into repeatable workflows (e.g., AI research into personalized outreach). Proactively shares workflows with teammates. |
| **Transformative** | Redesigned a core motion (prospecting, qualification) around AI-native logic. AI-driven workflows have produced measurable team-level impact (e.g., increased pipeline coverage). |

---

## People

| Level | Behaviors |
|-------|-----------|
| **Unacceptable** | Manually does work AI could assist with (comp modeling, scenario planning). Actively blocks the team from experimenting with AI. |
| **Capable** | Uses AI daily with repeatable prompt templates. Connected AI tools into recurring workflows (auto-summaries into status updates). |
| **Adoptive** | Automated a core People process end-to-end (agentic screening, AI-generated deliverables). Replaced manual work with data compilation and predictive modeling. |
| **Transformative** | Stopped running a legacy program entirely and rebuilt it around AI-first delivery. Redefined team roles around the new operating model. |
